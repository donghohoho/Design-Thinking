# 프로젝트 컨텍스트: Homepage

이 문서는 Gemini CLI가 프로젝트의 목적, 기술 스택, 개발 규칙을 이해하고 일관된 지원을 제공하기 위해 작성된 지침서입니다.

## 1. 프로젝트 개요 (Project Overview)
현재 이 디렉토리는 `Homepage`라는 이름의 새로운 프로젝트를 시작하기 위한 초기 상태입니다.

- **목적:** 개인 홈페이지 또는 서비스의 메인 페이지 개발.
- **주요 기술 스택 (예정):** 
  - 프론트엔드: React (TypeScript) 또는 Vanilla CSS/JS 기반 웹 기술.
  - 백엔드: Node.js (Express) 또는 FastAPI (필요 시).
  - 스타일링: Vanilla CSS (사용자 선호도에 따라 조정 가능).
- **아키텍처:** 초기 프로토타입 단계로, 시각적으로 완성도 높은 프론트엔드 중심의 개발이 예상됩니다.

## 2. 빌드 및 실행 (Building and Running)
현재 프로젝트 파일이 존재하지 않으므로, 아래는 일반적인 웹 프로젝트를 위한 가이드라인입니다.

- **설치:** `npm install` (Node.js 기반인 경우)
- **실행:** `npm run dev` 또는 `npm start`
- **테스트:** `npm test`
- **빌드:** `npm run build`

## 3. 개발 규칙 및 컨벤션 (Development Conventions)

### 3.1. 기본 원칙
- **사용 언어:** 모든 설명 및 주석은 기본적으로 한국어를 사용합니다.
- **최신성 유지:** 2026년 기준의 최신 기술 트렌드와 정보를 반영하여 개발합니다.
- **단계별 사고:** 문제 해결 시 논리적인 단계를 거쳐 접근하고 근거를 명확히 합니다.

### 3.2. Git 사용 규칙
- **브랜치:** 기본 작업 브랜치는 `dongho`를 사용합니다.
- **커밋 메시지 형식:** `[타입] 작업 내용 요약` 형식을 준수합니다.
  - 타입 예시: `[Feat]`, `[Fix]`, `[Design]`, `[Refactor]`, `[Chore]` 등.
- **문서화:** 모든 코드나 기능 변경 사항은 항상 `README.md`에 즉각적으로 반영해야 합니다.

### 3.3. 코드 스타일
- **프론트엔드:** 시각적 효과와 사용자 경험(UX)을 중시하며, 깔끔하고 유지보수가 용이한 코드를 작성합니다.
- **보안:** API 키나 민감한 정보는 절대 코드에 포함하거나 커밋하지 않습니다. `.env` 파일을 활용하십시오.

## 4. 주요 파일 및 디렉토리 구조 (Key Files)
- `src/App.tsx`: 핵심 상담 로직 (표정 분석, 음성 인식, Gemini AI 연동)
- `src/App.css`: 상담사 인터페이스 스타일링
- `src/main.tsx`: 앱 진입점
- `index.html`: 메인 HTML 템플릿
- `package.json`: 의존성 관리 및 스크립트
- `.env`: API 키 등 환경 변수 (로컬 전용)
